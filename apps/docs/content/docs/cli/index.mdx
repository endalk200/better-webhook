# CLI Getting Started

The new Better Webhook CLI release line ships as `bw`.

This page covers installation, release channels, and version verification for the current `2.x` CLI rollout.

## Installation

### Homebrew (stable)

```bash
brew install --cask endalk200/tap/better-webhook
```

### npm (stable)

```bash
npm install -g @better-webhook/cli
```

### npm prerelease testing

Prerelease testing should prefer `npx` instead of a global install:

```bash
npx @better-webhook/cli@next --version
npx @better-webhook/cli@2.0.0-alpha.1 version --json
```

### GitHub Releases

Download release assets from [GitHub Releases](https://github.com/endalk200/better-webhook/releases).

Expected archive names:

| Platform      | Asset Name               |
| ------------- | ------------------------ |
| macOS (ARM)   | `bw-darwin-arm64.tar.gz` |
| macOS (Intel) | `bw-darwin-x64.tar.gz`   |
| Linux (ARM)   | `bw-linux-arm64.tar.gz`  |
| Linux (x64)   | `bw-linux-x64.tar.gz`    |
| Windows (x64) | `bw-windows-x64.zip`     |

### Nix

Stable installs can build from source using the repo flake:

```bash
nix run github:endalk200/better-webhook?ref=cli-v2.0.0#bw -- --version
```

## Verify installation

Human-readable version:

```bash
bw --version
```

Machine-readable version metadata:

```bash
bw version --json
```

Example JSON fields:

- `schemaVersion`
- `version`
- `commit`
- `date`
- `platform`

## Release behavior

- Official CLI releases publish only from `cli-v*` tags.
- Stable releases publish to GitHub Releases, npm `latest`, Homebrew, and Nix validation.
- Prereleases publish to GitHub prereleases, npm `next`, and Nix validation.
- Homebrew stays stable-only in the first version.

## Artifact verification

GitHub Releases are the canonical source for:

- archives
- checksums
- Sigstore bundles
- SBOMs

For maintainer details, see [Release Policy](/docs/release-policy) and the internal runbook in `docs/cli-release-runbook.md`.
